# CSS_ARCHITECTURE_RULES.md
**Строгая архитектура CSS для Cursor (v1)**

## Назначение
Этот документ определяет, **КУДА и КАК** Cursor может добавлять стили.
Нарушение архитектуры считается ошибкой.

---

## 1. Структура CSS-файлов (ОБЯЗАТЕЛЬНО)

Cursor обязан использовать **только существующие файлы**:

- `base.css`
  – reset  
  – CSS-переменные  
  – типографика по тегам (`h2–h4`, `p`)

- `layout.css`
  – контейнер  
  – grid / row / col  
  – `.section` (отступы секций)

- `components.css`
  – стили отдельных блоков и компонентов  
  – только классы компонентов

- `pages/*.css`
  – стили, относящиеся ТОЛЬКО к конкретной странице  
  – допускается 1 уровень контекста страницы

❌ Запрещено:
- создавать новые CSS-файлы  
- переносить стили между файлами без команды  
- складывать всё в один файл  

---

## 2. Принцип «один селектор — одно место»

- Каждый селектор определяется **только один раз**
- Если селектор уже существует:
  → **дополняй его**
  → **не создавай новый**

❌ Запрещено:
- дублировать один и тот же класс в разных местах  
- переопределять класс в нескольких файлах  

---

## 3. Порядок стилей внутри файла (КРИТИЧНО)

1. Базовые стили блока  
2. Элементы блока  
3. Модификаторы (если разрешены)  
4. Media-запросы (строго в конце)

❌ Запрещено:
- вставлять media-запросы между стилями  
- разбивать стили одного блока по файлу  

---

## 4. Порядок блоков = порядок на странице

- Стили идут **в том же порядке**, что и блоки на странице
- Каждый блок — **цельная секция стилей**

❌ Запрещено:
- хаотичное расположение классов  
- «прыжки» по файлу  

---

## 5. Контекст и вложенность

- Контекстные селекторы — **только при необходимости**
- Максимум — **1 уровень вложенности**

✅ Допустимо:
```css
.course-card .title {}
```

❌ Запрещено:
```css
.page-course .section .course-card .title {}
```

---

## 6. Наследование и повторы (ОЧЕНЬ ВАЖНО)

- Если у родителя уже задано свойство:
  → **не повторяй его в дочернем элементе**

Особенно запрещено дублировать:
- цвет  
- шрифт  
- размер текста  
- `line-height`  

Если визуально одинаково — **стиль не нужен**.

---

## 7. Media-запросы (кратко)

- Все media — **в конце файла**
- Один брейк = один `@media`
- Брейкпоинты строго:
  - 1400  
  - 1280  
  - 768  

Порядок:
```
desktop → tablet → mobile
```

❌ Запрещено:
- повторять одинаковые `@media`  
- дробить стили блока по разным брейкам  

---

## 8. ИМЕНОВАНИЕ КЛАССОВ (ОБЯЗАТЕЛЬНО)

Используется упрощённый BEM:
– block
– block__element
– block--modifier

Запрещено:
– произвольные имена классов
– camelCase и PascalCase
– классы по визуальному признаку (big, red, left и т.п.)
– вложенные BEM-цепочки (block__el__subel)

Если требуется модификатор — он добавляется ТОЛЬКО по явной команде пользователя.

---

## 9. Если непонятно, куда писать стиль

Если Cursor сомневается:
- base / layout / component / page

→ **остановись и задай вопрос**  
→ **не принимай решение самостоятельно**
