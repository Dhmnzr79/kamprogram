# GRID_RULES.md
**12-колоночная сетка — строгие правила для Cursor (v1)**

```
НАЗНАЧЕНИЕ
Этот документ определяет ЕДИНСТВЕННЫЕ правила работы с сеткой.
Сетка — это ИНФРАСТРУКТУРА, а не дизайн.
```

---

## 1. Базовая модель сетки (ОБЯЗАТЕЛЬНО)

- Используется **12-колоночная grid-сетка**
- Контейнер, `.row`, `.col-*` — уже существуют
- Cursor НЕ ИМЕЕТ ПРАВА менять реализацию сетки

Брейкпоинты проекта (фиксированные):
- 768px — мобилка
- 1280px — планшет
- 1400px — десктоп

---

## 2. Назначение сетки (очень важно)

Grid-сетка отвечает **ТОЛЬКО** за:
- ширину колонок
- расположение колонок в строке

Grid-сетка **НЕ отвечает** за:
- вертикальные отступы секций
- внутренние отступы карточек
- визуальный ритм
- дизайн

Любые отступы — **не часть сетки**.

---

## 3. Формат работы с сеткой

### Cursor МОЖЕТ:
- использовать `.row`
- использовать `.col-*`, `.col-sm-*`, `.col-lg-*`
- использовать готовые паттерны сетки

### Cursor НЕ МОЖЕТ:
- добавлять inline-стили
- добавлять margin / padding
- менять gap сетки
- добавлять собственные grid / flex
- менять существующие классы сетки

---

## 4. Режим использования (ОБЯЗАТЕЛЬНО)

Все операции с сеткой выполняются:

```
ТОЛЬКО в режиме: HTML_ONLY
```

В режиме `STYLE_ONLY` и `CHECK_ONLY`:
- сетка не трогается
- HTML не меняется

---

## 5. HTML-only правило (жёстко)

При работе с сеткой Cursor обязан выводить:

- ТОЛЬКО HTML
- ТОЛЬКО `.row` и дочерние `.col-*`
- БЕЗ:
  - обёрток
  - inline-стилей
  - дополнительных div’ов
  - комментариев (кроме названия секции, если разрешено)

---

## 6. Каталог паттернов (обязателен к использованию)

Cursor обязан использовать **ГОТОВЫЕ паттерны**, а не придумывать новые.

### Основные паттерны

- **201** — 8 / 4  
- **202** — 4 / 8  
- **203** — 8 / 4 (смена порядка на мобилке)  
- **204** — 6 / 3 / 3  
- **205** — 4 / 4 / 4  
- **206** — 8 / 4 (на планшете 6 / 6)  
- **207** — 8 + offset 2  
- **208** — 3 / 3 / 3 / 3  
- **209** — 6 / 6  
- **210** — 12  

Композитные паттерны:
- **301** — две строки: 8/4 + 4/4/4

---

## 7. Запрет на отклонения от паттернов (КРИТИЧНО)

Cursor **НЕ ИМЕЕТ ПРАВА**:

- изменять схему колонок
- добавлять или убирать колонки
- комбинировать несколько паттернов
- менять порядок колонок без указания
- «оптимизировать» сетку
- подгонять под контент

Если паттерн не подходит —  
**Cursor обязан задать вопрос**, а не менять сетку.

---

## 8. Мобильное поведение (фиксировано)

Если явно не указано иное:

- на мобилке все колонки = **12 / 12**
- порядок колонок — как в HTML
- никаких автоматических перестановок

Любое изменение порядка:
- **только по явной команде пользователя**
- через готовые utility-классы (если они существуют)

---

## 9. Связь с другими правилами

- Grid-правила подчиняются `CURSOR_CORE_RULES.md`
- Отступы регулируются **ТОЛЬКО** через `SPACING_DEFAULTS.md`
- Архитектура CSS — через `CSS_ARCHITECTURE_RULES.md`

Grid-файл **НЕ расширяет** дизайн-возможности Cursor.

---

## 10. Поведение при сомнениях (обязательно)

Если:
- паттерн не указан
- схема неочевидна
- макет не укладывается в паттерн

→ Cursor останавливается  
→ не генерирует HTML  
→ задаёт вопрос пользователю
